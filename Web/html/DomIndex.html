Preguntas

    1. ¿Qué es el DOM?

      • ChatGPT: Es una representación estructurada jerárquica de un documento HTML o XML que proporciona una interfaz para interactuar y manipular dinámicamente elementos de la página web.

      • Otras fuentes: Segun Mozilla DOM significan Document Object Model, o lo que es lo mismo, la estructura del documento HTML. Una página HTML está formada por múltiples etiquetas HTML, anidadas una dentro de otra, formando un árbol de etiquetas relacionadas entre sí, que se denomina árbol DOM (o simplemente DOM). En Javascript, cuando nos referimos al DOM nos referimos a esta estructura, que podemos modificar de forma dinámica desde Javascript, añadiendo nuevas etiquetas, modificando o eliminando otras, cambiando sus atributos HTML.

    2. ¿Qué es un nodo?

      • ChatGPT: Un nodo representa un elemento individual en una estructura de árbol que representa un documento HTML.

      • Otra fuente: Segun Keepcoding el nodo en el DOM es cada una de las etiquetas HTML del árbol jerárquico.

        Accceso por indentificador, con el metodo "getElmentById()"

        Accceso por etiquetas, con el metodo "getElmentByTagName()"

        Accceso por clases , con los siguientes metodos:
        "querySelector()" Este metodo devuelve solo el primer elemento que cumple con el selector

    4. ¿Cómo se modifica un elemento del DOM?

      • ChatGPT: Obtén una referencia al elemento que deseas modificar. Puedes utilizar métodos como getElementById, getElementsByClassName, getElementsByTagName, querySelector, etc., para seleccionar el elemento deseado. Estos métodos devuelven una referencia al elemento que coincida con el selector proporcionado.

        Utiliza las propiedades y métodos disponibles para modificar el elemento según tus necesidades. Algunas de las propiedades comunes son:

        • innerHTML: Permite modificar el contenido HTML del elemento.
        • textContent: Permite modificar el contenido de texto del elemento.
        • setAttribute: Permite modificar atributos del elemento, como src, href, class, etc.
        • style: Permite modificar los estilos CSS del elemento, como el color, tamaño de fuente, etc.
        • classList: Permite modificar las clases del elemento, agregando o quitando clases.

        Por ultimo asigna los nuevos valores a las propiedades o utiliza los métodos correspondientes para realizar las modificaciones deseadas en el elemento.

      • Otra fuente: Según Mozilla la manera mas fácil de modificar los elementos del DOM, seria con la función
        Document.”elemento a modificar”.innerHTML = “”;

    5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

      • ChatGPT: La principal diferencia entre getElementBy* y querySelector* es el tipo de selección que permiten realizar. getElementBy* permite seleccionar elementos del DOM por su ID, su clase o su etiqueta, mientras que querySelector* permite seleccionar elementos utilizando selectores CSS, lo que ofrece una gran flexibilidad para realizar selecciones más precisas y complejas.

      • Otra fuente: getElementByid permite recuperar un elemento del modelo de objetos del DOM. Solamente es aplicables a los elementos que este tenga su selector como ID.
        Mientras que querySelector, puede seleccionar un elemento basado en un selector de CSS, es decir puede tomar elementos de ID, clases o cualquier selector.

    6. ¿Qué es un evento?

      • ChatGPT: Los eventos permiten que un programa JavaScript responda y realice acciones específicas cuando ocurre una interacción con el usuario o cuando se producen ciertos cambios en la página.

      • Otra fuente: En Javascript existe un concepto llamado evento, que no es más que una notificación de que alguna característica interesante acaba de ocurrir, generalmente relacionada con el usuario que navega por la página.

    7. ¿Qué es un listener?

      • ChatGPT: En el contexto de la programación, un listener (o "escuchador" en español) se refiere a una función o bloque de código que está diseñado para "escuchar" o "vigilar" eventos específicos que ocurren en un programa o sistema.

      • Otra fuente: Un escuchador de eventos en Java está diseñado para procesar algún tipo de evento “escucha” un evento, como el clic del ratón de un usuario o la pulsación de una tecla, y luego responde en consecuencia. Un escuchador de eventos debe estar conectado a un objeto de evento que define el evento.

    8. ¿Qué es un callback?

      • ChatGPT: es una función que se pasa como argumento a otra función y que se invoca dentro de esa función en un momento posterior. El propósito del callback es permitir la ejecución de código de manera asíncrona, es decir, que no se ejecute de forma secuencial en un determinado orden.

      • Otra fuente: es una función que se pasa a otra función como un argumento, que luego se invoca dentro de la función externa para completar algún tipo de rutina o acción.

    9. ¿Qué es un objeto de evento?

      • ChatGPT: un objeto de evento es un objeto especial que se crea automáticamente cuando ocurre un evento. Este objeto contiene información sobre el evento y proporciona métodos y propiedades para interactuar con él.

      • Otra fuente: Cuando se produce un evento, no es suficiente con asignarle una función responsable de procesar ese evento. Normalmente, la función que procesa el evento necesita información relativa al evento producido: la tecla que se ha pulsado, la posición del ratón, el elemento que ha producido el evento, etc.
        El objeto event es el mecanismo definido por los navegadores para proporcionar toda esa información. Se trata de un objeto que se crea automáticamente cuando se produce un evento y que se destruye de forma automática cuando se han ejecutado todas las funciones asignadas al evento.

    Conclusiones

    Experiencia general

    ¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

    Usuaria más ChatGPT, cuando apenas estoy iniciando a aprender nuevos lenguajes o nuevas funciones para programar, en lo cual me da más teoría en la cual pueda entender bien el lenguaje.

    Mientras de copillot lo usuaria cuando tenga dudas más concretas sobre líneas de código, también como estructura el código.

    Ventajas de usar AI para programar

    La facilitación de realizar programas en los cuales no tengamos un conocimiento suficiente, para poder reducir el tiempo para realizarlo. Además de poder aprender sobre la marcha.

    Desventajas de usar AI para programar

    Que en una buena parte de las veces la AI entregan códigos en los cuales no realizan adecuadamente la funcionalidad deseada y a veces no funcionan. Además, no procura que los códigos sean óptimos, teniendo complejidades de ejecución alta.
